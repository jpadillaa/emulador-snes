Resumen del Proyecto

Diseño y desarrollo de una aplicación de escritorio nativa para macOS construida en Python. La aplicación actúa como una capa de presentación y gestión sobre un núcleo de emulación de Super Nintendo de terceros. El diseño prioriza la separación estricta entre la lógica de interfaz de usuario y el procesamiento de la emulación garantizando alta modularidad y bajo acoplamiento.

Alcance

Dentro del Proyecto

  • Interfaz gráfica nativa macOS
  • Gestión de biblioteca de ROMs
  • Configuración de controles y periféricos
  • Persistencia de partidas guardadas
  • Gestión de video y audio
  • Puente de integración (Adaptador)

Fuera del Proyecto

  • Implementación de la emulación de hardware
  • Procesamiento de señales de bajo nivel
  • Desarrollo de motores de renderizado 3D

Riesgos y Dependencias

  • Dependencia absoluta de la API del núcleo de emulación externo seleccionado
  • Posible latencia introducida por el puente de comunicación entre Python y C/C++
  • La implementación del Adaptador del Núcleo está sujeta a si el núcleo externo es una biblioteca compartida o un proceso independiente
  • Actualizaciones en las políticas de seguridad de macOS (Gatekeeper) para captura de periféricos Bluetooth

Requerimientos Funcionales

  • Analizar directorios locales para catalogar ROMs válidas
  • Presentar biblioteca interactiva con metadatos y arte
  • Proveer interfaz para configuración de controles y teclado
  • Gestionar persistencia de estados de guardado
  • Configurar parámetros de salida multimedia
  • Capturar y renderizar búferes del núcleo de emulación

Requerimientos No Funcionales

  • Mantenibilidad Uso de segregación de interfaces y patrón adaptador
  • Modularidad Componentes independientes con responsabilidades únicas
  • Rendimiento Latencia imperceptible en la capa de integración de Python
  • UX Alineación con estándares de diseño macOS integrando aceleración por hardware

Arquitectura del Sistema

Capa de Presentación

Renderiza la interfaz gráfica y la superficie de video. Recibe acciones del usuario e interactúa con el Controlador.

Gestor de Biblioteca

Escanea archivos locales, calcula firmas, obtiene metadatos y maneja el inventario en la base de datos local.

Gestor de Configuración

Centraliza y persiste preferencias de video, audio y atajos de teclado del usuario.

Gestor de Entrada

Detecta dispositivos Bluetooth, captura teclado y mapea eventos físicos a botones virtuales de SNES.

Adaptador del Núcleo (Bridge)

Aísla Python de la implementación del núcleo. Transfiere rutas ROM, estado de controles, comandos y extrae búferes multimedia y estados de guardado sin acoplar la UI al emulador base.

Flujo de Usuario Principal

  1. 1

    Inicio

    El SO inicia la app. El Controlador carga configuraciones previas.

  2. 2

    Carga de Biblioteca

    El Gestor de Biblioteca consulta BD y renderiza cuadrícula asíncrona.

  3. 3

    Ejecución

    El usuario selecciona ROM. El Adaptador inicializa el núcleo externo.

  4. 4

    Interacción Continua

    El Gestor de Entrada envía pulsaciones. El Adaptador extrae frames para renderizado Metal.

  5. 5

    Interrupción

    Usuario envía comando de pausa/guardado. El núcleo exporta estado a disco.
